# news-md｜韩语新闻正文提取器

[English](README.en.md)

## 项目简介

`news-md` 是一个轻量级本地 CLI 工具，用来把单条韩语新闻链接整理成相对干净、可复核的 Markdown 文本。

它服务的是个人内容研究流程：先把网页中的新闻正文提取出来，减少广告、推荐新闻、版权说明、记者邮箱、站点导航等明显噪声，再把结果用于后续 AI 翻译、摘要、内容分析或归档复核。

这个项目不是大型平台，也不是通用爬虫。它的重点是把一个真实、重复、容易出错的整理步骤，做成一个边界清楚的小工具。

## 为什么做这个工具

### 1. 既然有 AI，为什么不直接丢链接？

直接把新闻链接交给 AI 并不总是稳定。新闻网页可能存在跳转、SSL 问题、动态加载、访问限制，或者正文周围混有大量页面模块，AI 工具不一定能稳定拿到正文。

### 2. 既然新闻网页能打开，为什么不直接复制？

手动复制正文也很低效。复制出来的内容经常夹带推荐列表、版权块、记者信息、站点导航或重复段落，后续还要再清理一遍。

### 3. 既然有现成工具，为什么还要自己开发？

我没有找到专门面向韩网新闻正文整理、并能接入本地研究流程的轻量工具。通用抽取工具可以提供基础正文抽取能力，但还需要结合韩网新闻页面的噪声特征、元数据保留和回归样本复核。

因此，`news-md` 选择先做一件小事：把单条新闻网页整理成 Markdown，让后续的翻译、摘要和阅读分析有一个更稳定、可检查的输入。

## 它解决的问题

- 减少手动复制和清理新闻正文的重复劳动。
- 把网页内容转换成更适合阅读、保存和交给 AI 处理的 Markdown。
- 通过基础元数据和质量状态，提醒使用者检查标题、来源、发布时间和正文完整性。
- 保留人工复核环节，避免把自动抽取结果当作最终权威文本。

## 工作流程

```text
韩语新闻 URL
  -> 正文抽取
  -> 明显噪声清理
  -> Markdown 输出
  -> AI 翻译 / 摘要 / 内容研究 / 归档复核
```

## 主要功能

- 接收单条韩语新闻 URL。
- 抽取类似新闻正文的内容。
- 尽量清理明显页面噪声。
- 输出 Markdown 文件。
- 记录标题、来源、发布时间、抽取方式、正文质量状态等基础信息。
- 支持通过回归样本和人工复核记录观察抽取效果。

## 项目亮点

- **问题定义清楚**：从个人内容研究中的真实重复步骤出发，没有把项目扩大成不必要的平台。
- **工具边界克制**：只处理“单条 URL 到 Markdown”这个核心流程，不承担监控、翻译、服务端托管等职责。
- **结果可复核**：输出 Markdown 和质量状态，方便人继续检查，而不是假设自动化结果一定正确。
- **验证意识明确**：用多批真实样本做过验证，并用 pass / partial / problem / failed 等类别记录结果。
- **公开边界清晰**：公开文档只保留聚合总结和限制说明，不公开真实 URL 列表、原始日志或内部复核材料。

## 使用方式

该工具面向 Windows / PowerShell 本地使用，可以通过批处理入口运行，也可以按本地 CLI 提示输入新闻链接。

示例：

```powershell
news-md "https://example.com/korean-news-article"
```

输出结果默认保存为 Markdown，具体输出位置可通过命令参数或本地环境配置调整。

## 当前边界

`news-md` 适合作为个人内容研究和作品集展示中的轻量工具，但不应被理解为正式发布的通用产品。

它不是：

- 通用爬虫平台
- 新闻监控系统
- 自动翻译系统
- dashboard
- server / hosted service
- 面向所有网站和所有版式的完整抽取方案

它也不保证每篇韩语新闻都能完美抽取。对于重要材料，仍建议对照原网页进行人工复核。

## 技术栈

- Python
- Windows batch / PowerShell 本地运行方式
- `url2md4ai` 作为主要抽取方式
- `trafilatura` fallback
- BeautifulSoup 元数据解析
- Markdown 输出与质量状态标记
