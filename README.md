# news-md｜韩语新闻正文提取器

[English](README.en.md)

## 项目简介

`news-md` **是一个轻量级本地 CLI 工具，用来把单条韩语新闻链接整理成相对干净、可复核的 Markdown 文本。**

## 为什么做这个工具

这个项目来自我的内容研究流程：在阅读和整理韩语新闻时，我经常需要先把网页正文提取出来，再交给 AI 做翻译、摘要或内容分析。

**1. 既然有 AI，为什么不直接丢链接？**

因为新闻网页不只是正文。页面可能有跳转、SSL 问题、动态加载、访问限制，也可能在正文周围混入大量推荐模块和站点内容。AI 工具并不总能稳定读取到真正需要的文章正文。

**2. 既然新闻网页能打开，为什么不直接复制？**

手动复制可以解决一次问题，但不适合反复做。复制出来的内容经常夹带广告、推荐列表、版权块、记者邮箱、站点导航或重复段落，后续还要再清理一遍，效率低，也不容易复核。

**3. 既然有现成工具，为什么还要自己开发？**

通用正文抽取工具可以提供基础能力，但我的工作流还需要更贴近韩语新闻页面的噪声处理、元数据保留、Markdown 输出和样本回归复核。`news-md` 不是为了重新造一个大平台，而是把这些需求收束成一个轻量、本地、可控的小工具。

`news-md` 解决的是一个很小但高频的问题：把单条新闻网页先整理成 Markdown，让后续处理有一个更可靠的输入。它不是大型平台，也不是通用爬虫，而是一个边界清楚的本地工作流工具。

## 工作流程

```text
韩语新闻 URL
  -> 正文抽取
  -> 明显噪声清理
  -> Markdown 输出
  -> AI 翻译 / 摘要 / 内容研究 / 归档复核
```

## 主要功能

- 接收单条韩语新闻 URL。
- 抽取类似新闻正文的内容。
- 尽量清理明显页面噪声。
- 输出 Markdown 文件。
- 记录标题、来源、发布时间、抽取方式、正文质量状态等基础信息。
- 支持通过回归样本和人工复核观察抽取效果。

## AI 协作开发方式

这个项目也是一次 AI 协作开发实践，但重点不是把它包装成“智能平台”，而是把 AI 用在具体、可检查的工作环节里。

- GPT 用于需求拆解、项目边界判断、README 结构、验证思路和公开材料取舍讨论。
- Codex 用于代码修改、局部修复、冲突处理、GitHub 上传准备和文档整理。
- 我自己负责判断需求是否真实、输出是否可用、项目范围是否开始膨胀，以及哪些材料适合公开。

## 项目亮点

- **真实问题识别**：从韩语新闻内容研究中的高频重复步骤出发，而不是先想一个抽象的 AI 项目概念。
- **轻量工具设计**：只围绕“单条 URL 到 Markdown”设计流程，用 CLI 降低重复操作成本。
- **AI 协作开发**：用 GPT / Codex 辅助拆解、修改、排错和整理，同时保留人工判断。
- **测试与人工复核**：用多批样本做验证，并按 pass / partial / problem / failed 等类别观察输出质量。
- **public/private 边界意识**：公开文档只保留聚合总结和限制说明，不公开真实 URL 列表、原始日志或内部复核材料。

## 使用方式

该工具面向 Windows / PowerShell 本地使用，可以通过批处理入口运行，也可以按本地 CLI 提示输入新闻链接。

示例：

```powershell
news-md "https://example.com/korean-news-article"
```

输出结果默认保存为 Markdown，具体输出位置可通过命令参数或本地环境配置调整。

## 技术栈

- Python
- Windows batch / PowerShell 本地运行方式
- `url2md4ai` 作为主要抽取方式
- `trafilatura` fallback
- BeautifulSoup 元数据解析
- Markdown 输出与质量状态标记
